# Commit Message Format

This project uses freeform commit messages. Follow these conventions:

Format: Short imperative description, sentence case, no prefix
Case: Sentence case (capitalize first word only)
Scopes: none

Examples from this project:
- "Add kitty and starship configs, update zsh env"
- "Initial dotfiles setup: modular zshrc with conditional loading"
- "Updated startday and fd to have the -u --glob and --full-path by default"
